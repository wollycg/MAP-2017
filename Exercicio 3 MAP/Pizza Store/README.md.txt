Uma interface é como uma classe 100% abstrata. Uma interface é como um conjunto de regras (contrato) definidas e que devem ser rigorosamente seguidas (implementadas). Com ele nao precisamos saber como é feito, somente o que ela faz.Interface permite um polimorfismo independente da hierarquia. Permitindo criar interfaces sem que a classe concreta seja especificada. No exemplo, o cliente vai receber todos os sabores disponiveis quando chamar as respectivas pizzarias New York ou de Chicago.
A herança possibilita o reaproveitamento de código, onde cada subclasse herda as características da super classe.Mas quando mudar em um nao tem q sair mudando em outros locais.Gerando um alto acoplamento.
